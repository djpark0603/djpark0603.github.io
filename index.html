About

지금 필요한 최소 구성을 먼저 올렸습니다.

왜 이렇게 시작했는가

GitHub Pages는 복잡한 서버 없이도 빠르게 공개할 수 있습니다. 그래서 첫 버전은 소개, 링크, 향후 업데이트 자리만 명확하게 잡아두는 방식으로 구성했습니다.

이후 바로 바꾸기 좋은 부분

소개 문구, 프로젝트 카드, 기술 스택, 외부 링크만 수정하면 바로 자신만의 포트폴리오 사이트로 확장할 수 있습니다.

Focus

현재 사이트에 넣어둔 핵심 블록

소개

방문자가 첫 화면에서 누구의 페이지인지 바로 알 수 있도록 이름, GitHub ID, 연락 수단을 묶었습니다.

업데이트 기반

프로젝트 경험이 늘어나면 카드만 추가해도 구조가 깨지지 않도록 반응형 레이아웃으로 잡았습니다.

배포 단순성

순수 HTML/CSS/JS만 사용해서 GitHub Pages에 바로 올릴 수 있게 했습니다. 별도 빌드 도구가 필요 없습니다.
